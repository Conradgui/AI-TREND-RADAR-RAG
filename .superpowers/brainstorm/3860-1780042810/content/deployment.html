部署流程

用户本地部署的完整步骤

一键部署流程

# 1. 克隆项目
git clone https://github.com/Conradgui/AI-TREND-RADAR.git
cd AI-TREND-RADAR

# 2. 配置 API Key（复用同一份 .env）
cp .env.example .env
# 编辑 .env 填入 LLM_PROVIDER 和对应的 API_KEY

# 3. 安装依赖
pnpm install                # TypeScript 依赖
pip install -r rag/requirements.txt  # Python RAG 依赖

# 4. 启动 Neo4j（Docker）
docker run -d --name neo4j \
  -p 7474:7474 -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password \
  neo4j:5

# 5. 运行数据管道 + 摄取
pnpm digest                # 生成日报数据
python -m rag.ingest        # 摄取到 Neo4j + ChromaDB

# 6. 启动 RAG 服务
python -m rag.server        # http://localhost:8001

# 7. 打开浏览器对话
→ http://localhost:8001

文件隔离设计

AI-TREND-RADAR/
├── src/              ← 现有 TS 管道（不变）
├── digests/           ← 日报数据（共享）
├── rag/               ← 新增 RAG 系统（独立目录）
│  ├── server.py       FastAPI 服务入口
│  ├── ingest.py       摄取脚本入口
│  ├── config.py       读取项目 .env
│  ├── ingestion/      数据摄取层
│  ├── graph/          Neo4j 图数据库层
│  ├── retriever/      检索层（向量+图混合）
│  ├── agent/          Agent + 工具
│  ├── web/            Chat UI 前端
│  ├── data/           持久化数据（ChromaDB 等）
│  ├── requirements.txt
│  └── pyproject.toml
├── docker-compose.yml  ← 一键启动 Neo4j
└── .env               ← 共用 API 配置
