Chat 交互流程

用户通过右侧面板与 AI Agent 对话的完整流程

用户操作流程

1. 用户打开 http://localhost:8001
2. 看到报告界面（和 GitHub Pages 上一样）
3. 点击 header 的 "🤖 Agent" 按钮
4. 右侧面板滑出（宽度 380px）
5. 输入问题："最近 RAG 有什么新进展？"
6. Agent 自动：
  a. 在 topic-pool.json 中搜索相关话题
  b. 在 Neo4j 知识图谱中查询相关实体和关系
  c. 调用 LLM 综合分析并生成回答
7. 回答显示在面板中（支持 markdown 渲染）
8. 引用来源可点击，跳转到对应日期的报告

Agent 工具系统（4 个工具）

graph_search

Neo4j 知识图谱查询：话题关系、实体网络、跨时间趋势

vector_search

ChromaDB 语义搜索：在所有日报/周报/月报中按语义相似度查找相关内容

trend_analysis

话题趋势分析：某个话题在不同日期的分数变化、热度走向

topic_recommend

选题推荐：基于历史数据和当前趋势，推荐值得深挖的选题

数据流

用户输入 → FastAPI /chat 端点
  ↓
LangGraph ReAct Agent
  ↓ 判断需要什么工具
  ├─ graph_search → Neo4j Cypher 查询
  ├─ vector_search → ChromaDB 语义搜索
  ├─ trend_analysis → Neo4j 时间序列查询
  └─ topic_recommend → 综合评分排序
  ↓
拼接 context + 用户问题 → 调用 LLM
  ↓
返回 {answer, citations} → 浏览器渲染
