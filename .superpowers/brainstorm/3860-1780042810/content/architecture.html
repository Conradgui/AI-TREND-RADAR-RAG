整体架构

TypeScript 数据管道 + Python RAG 后端，本地部署

系统架构图
📁 AI-TREND-RADAR (现有)
pnpm digest
├─ fetchAllData (GitHub/HN/ArXiv/36kr...)
├─ generateSummaries (LLM)
├─ saveTopicRadar
└─ 输出 → digests/YYYY-MM-DD/
    ├─ topic-pool.json
    ├─ ai-topic-radar.md
    └─ ai-*.md (各源报告)
→
🧠 RAG Backend (新增)
python -m rag.server (http://localhost:8001)
├─ 摄取层
│  ├─ topic_pool_loader (解析 JSON)
│  ├─ report_loader (解析 MD)
│  └─ neo4j-graphrag SimpleKGPipeline
│     └─ LLM 实体抽取 → 知识图谱
├─ 存储层
│  ├─ Neo4j (知识图谱 + 向量索引)
│  └─ ChromaDB (文档向量)
├─ 检索层
│  └─ HybridRetriever (向量+BM25+图)
├─ Agent 层
│  └─ LangGraph ReAct Agent
│     ├─ graph_search 工具
│     ├─ vector_search 工具
│     ├─ trend_analysis 工具
│     └─ topic_recommend 工具
└─ Web UI
    └─ 右侧滑出 Chat 面板
🐳 Docker Compose (一键启动)
docker-compose up -d
├─ neo4j:5 (图数据库, 端口 7687/7474)
└─ rag-server (Python FastAPI, 端口 8001)
